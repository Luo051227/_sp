## 程式一：銀行存款模擬

### 1.1 問題描述

模擬同一個銀行帳戶的多執行緒存提款操作：
- 初始存款：0 元
- 4 個執行緒執行存款
- 4 個執行緒執行提款
- 每個執行緒執行 100,000 次
- 每次金額：100 元

### 1.2 預期結果

由於存款次數和提款次數相同（各 400,000 次），且每次金額相同，最終餘額應為 0 元。

### 1.3 程式架構

```python
# 帳戶餘額
balance = 0

# 建立互斥鎖
lock = threading.Lock()
```

### 1.4 同步機制

使用 **threading.Lock** 保護共享的帳戶餘額：

```python
def deposit(thread_id):
    global balance
    
    for i in range(DEPOSIT_COUNT):
        with lock:  # 使用上下文管理器自動加鎖/解鎖
            balance += AMOUNT
```

### 1.5 競爭條件事說明

如果沒有互斥鎖，可能發生以下情況：

```
時間線：
T1: 執行緒A讀取餘額 (0)
T2: 執行緒B讀取餘額 (0)
T3: 執行緒A計算並寫入 (100)
T4: 執行緒B計算並寫入 (100)  <- 覆蓋了T3的結果！

結果：餘額 = 100（正確應該是 200）
```

## 程式二：生產者-消費者問題

### 2.1 問題描述

經典的執行緒同步問題：
- 緩衝區大小：10
- 2 個生產者，每個生產 100 個產品
- 2 個消費者，每個消費 100 個產品
- 生產者將產品放入緩衝區
- 消費者從緩衝區取出產品

### 2.2 同步機制

使用兩種同步機制：

1. **自定義訊號量（Semaphore）**
   - `empty_slots`：空位數量，初始為緩衝區大小
   - `full_slots`：產品數量，初始為 0

2. **互斥鎖（Mutex）**
   - 透過 queue.Queue 內建同步

### 2.3 訊號量實現

```python
class Semaphore:
    def __init__(self, initial_value):
        self._value = initial_value
        self._lock = threading.Lock()
        self._condition = threading.Condition(self._lock)
    
    def wait(self):
        with self._condition:
            while self._value <= 0:
                self._condition.wait()
            self._value -= 1
    
    def signal(self):
        with self._condition:
            self._value += 1
            self._condition.notify()
```

### 2.4 環形緩衝區

使用 queue.Queue 實現環形緩衝區：

```python
class CircularBuffer:
    def __init__(self, size):
        self.buffer = queue.Queue(maxsize=size)
        self.lock = threading.Lock()
    
    def put(self, item):
        self.buffer.put(item)
    
    def get(self):
        return self.buffer.get()
```

## 程式三：哲學家用餐問題

### 3.1 問題描述

經典的死結問題：
- 5 位哲學家圍坐在圓桌旁
- 每位哲學家左右各有一支叉子
- 只有拿到兩支叉子才能吃麵
- 吃完後放下叉子思考

### 3.2 問題分析

**可能發生的死結場景：**
```
所有哲學家同時拿起左邊的叉子
→ 每位哲學家都在等待右邊的叉子
→ 形成循環等待 → 死結
```

### 3.3 解決策略

本程式實現了**奇偶策略**來避免死結：

```python
# 奇數號哲學家：先拿左邊，再拿右邊
# 偶數號哲學家：先拿右邊，再拿左邊

if philosopher_id % 2 == 1:  # 奇數
    first, second = left_fork, right_fork
else:                        # 偶數
    first, second = right_fork, left_fork
```

**為何有效：**
- 打破了循環等待的條件
- 至少有一位哲學家不會與鄰居形成等待環

### 3.4 程式流程

```
哲學家迴圈（吃 MEALS_TO_EAT 餐）：
  1. 思考 (sleep)
  2. 嘗試拿起叉子
     - 先拿第一支叉子（根據策略）
     - 再拿第二支叉子
  3. 吃飯 (sleep)
  4. 放下叉子
```

### 3.5 叉子狀態表示

```python
self.forks = [False] * NUM_PHILOSOPHERS
# False = 可用
# True = 已被佔用
```

## Python執行緒同步機制比較

| 機制 | 用途 | 特性 |
|------|------|------|
| threading.Lock | 保護共享資源 | 独占鎖，只有lock的執行緒能unlock |
| threading.RLock | 遞迴鎖 | 同一執行緒可多次acquire |
| threading.Semaphore | 資源計數 | 可以計數，允許多個執行緒同時訪問 |
| threading.Condition | 等待特定條件 | 需要與lock配合使用 |
| queue.Queue | 執行緒安全隊列 | 內建同步機制 |
