## 習題 1：p0 編譯器

### 這在做什麼？

我們自己動手寫了一個**小型的編譯器**，它可以把簡單的程式碼翻譯成電腦能懂的指令，然後執行。

### 為什麼要學這個？

```
想象一下：
- 你寫了一個程式 (像寫文章)
- 電腦看不懂文章，只懂 0 和 1
- 所以需要一個「翻譯」把文章翻成 0 和 1

這個翻譯就是「編譯器」！
```

學會編譯器的原理，你會知道：
- 程式是怎麼被執行的
- 為什麼有些語言比較快、有些比較慢
- 以後學新的程式語言會更快

### 我們做了什麼功能？

| 功能 | 範例 | 說明 |
|------|------|------|
| 變數賦值 | `x = 10` | 把值存進記憶體 |
| 算術運算 | `a + b * c` | 加減乘除 |
| 比較運算 | `x < 5` | 大於、小於、等於 |
| if 判斷 | `if (x < 10) { ... }` | 條件分支 |
| **while 迴圈** | `while (x < 10) { ... }` | 重複執行 |
| 函數 | `func add(a, b) { return a + b }` | 重複使用的程式碼 |

### 核心概念：什麼是「四元組」？

編譯器先把程式轉換成一種中間格式，叫做**四元組**：

```
(運算碼, 運算元1, 運算元2, 結果)

範例：
(ADD, x, y, t1)     →  把 x + y 的結果存到 t1
(CALL, add, 2, t2)  →  呼叫 add 函數，傳 2 個參數，回傳值存到 t2
```

### 核心概念：函數呼叫機制

這是最重要的部分！函數呼叫是透過**堆疊幀 (Stack Frame)** 實現的。

```
生活中的比喻：
══════════════════════════════════
想象一家餐廳：
- 餐廳大廳 = 全域環境 (Frame 0)
- 包廂 = 函數環境 (Frame 1, 2, ...)

當你呼叫函數時，就像：
1. 進入一個包廂
2. 包廂裡的變數只屬於這個包廳
3. 離開包廂後，包廂裡的東西會被清空
4. 回到大廳繼續吃飯
```

**函數呼叫的步驟：**

```python
# 1. 呼叫時：建立新的 Frame
sp += 1  # 進入新環境
stack[sp] = Frame()
stack[sp].ret_pc = pc + 1  # 記住回來的位置
stack[sp].ret_var = "t3"   # 記住回傳值存哪

# 2. 執行函數內容...

# 3. 回傳時：銷毀 Frame，回到原環境
sp -= 1
pc = stack[sp].ret_pc  # 跳回原來的位置
```

### 動手練習

建立 `test.txt`：
```
x = 0;
while (x < 3) {
    x = x + 1;
}
```

執行：
```bash
python 1.py test.txt
```

## 習題 2：SimpleScript 直譯器

### 這在做什麼？

設計並實作了一個**完整的程式語言**！包含詞彙分析、語法分析、直譯器。

### 為什麼要學這個？

```
為什麼要自己設計語言？
══════════════════════════
1. 了解語言是怎麼運作的
   (Python、Java 是怎麼執行的？)

2. 學習解決問題的能力
   (如何把複雜的問題拆解成小問題)

3. 很好玩！
   (自己設計語言超有成就感！)
```

### 語言語法

```
SimpleScript 範例：
═══════════════════════════════════

// 變數宣告
let x: int = 10;
let name: string = "小明";

// 條件判斷
if x > 5 {
    print("大於5");
}

// 迴圈
for i = 0, 10, 1 {
    print(i);
}

// 函數
fn add(a: int, b: int) -> int {
    return a + b;
}
```

### 系統架構

```
原始程式碼 (.ss 檔案)
        │
        ▼
┌─────────────────┐
│  詞彙分析器     │  把字元切成 Token
│  (lexer.py)    │
└─────────────────┘
        │
        ▼ Token 流
┌─────────────────┐
│  語法分析器     │  檢查語法，建立語法樹
│  (parser.py)    │
└─────────────────┘
        │
        ▼ AST (語法樹)
┌─────────────────┐
│  直譯器        │  執行語法樹
│  (interpreter) │
└─────────────────┘
        │
        ▼
     輸出結果！
```

### 為什麼這樣分開？

```
比喻：翻譯一本書
═══════════════════════════

1. 詞彙分析 = 辨認每個字
   (這是「苹果」還是「蘋果」？)

2. 語法分析 = 檢查句子對不對
   (「我吃飯」是對的，「吃我飯」是錯的)

3. 直譯器 = 執行命令
   (按照句子的意思去做事)

分開做的好處：
- 比較好除錯
- 比較好擴充功能
- 程式碼比較整潔
```

## 習題 5：執行緒與同步

### 這在做什麼？

學習**多執行緒程式設計**，以及如何處理執行緒之間的同步問題。

### 為什麼要學這個？

```
現在的電腦都有多個核心！
═══════════════════════════
- 手機有 4 核、8 核
- 電腦有 4 核、8 核、16 核

如果只用一個核心，等於浪費其他核心！

多執行緒可以：
✓ 同時做多件事
✓ 提高程式速度
✓ 保持介面不會卡住
```

### 執行緒 vs 程序

```
比喻：
═══════════════════════════════

程序 (Process)  = 一間工廠
執行緒 (Thread) = 工廠裡的工人

- 一間工廠可以有很多工人
- 工人共享工廠的資源 (機器、原料)
- 但每個工人有自己的工作要做
```

| 特性 | 程序 | 執行緒 |
|------|------|--------|
| 資源 | 獨立空間 | 共享程序資源 |
| 建立成本 | 高 | 低 |
| 通訊 | 複雜 | 直接共享記憶體 |

### 競爭條件事 (Race Condition)

**什麼是競爭條件事？**

```
兩個執行緒同時改同一個變數！
═══════════════════════════

例子：銀行帳戶
- 帳戶原本有 1000 元
- 執行緒 A：存款 100 元 → 讀取 1000，加 100 = 1100
- 執行緒 B：提款 200 元 → 讀取 1000，减 200 = 800

如果同時執行：
A 讀取 1000
B 讀取 1000  (A 還沒寫回！)
A 寫入 1100
B 寫入 800  (覆蓋了 A 的結果！)

最終餘額：800 元 (應該是 900 元！)
```

### 互斥鎖 (Mutex)

**解決方案：加鎖！**

```python
import threading

lock = threading.Lock()
balance = 1000

def deposit(amount):
    with lock:  # 鎖住這段程式碼
        global balance
        balance += amount
```

```
比喻：廁所的門鎖
═══════════════════════════
- 有人進入廁所 → 鎖門
- 其他人只能在外面等
- 裡面的人出來 → 開門
- 下一個人才能進入
```

### 死結 (Deadlock)

**什麼是死結？**

```
大家互相等待對方！
═══════════════════

哲學家用餐問題：
- 5 位哲學家坐在圓桌
- 每個人需要左右兩把叉子才能吃飯
- 如果每個人都拿了左邊的叉子
- 大家在等右邊的叉子
- 結果：大家都在等，沒人可以吃！

這就是「死結」！
```

**如何避免死結？**

1. **固定順序**：總是先拿編號小的叉子
2. **一次拿完**：不要分兩次拿叉子
3. **打破循環**：讓某個人不一樣的順序拿

### 經典問題

| 問題 | 說明 | 解決方案 |
|------|------|----------|
| 銀行存款 | 多人同時存提款 | 互斥鎖 |
| 生產者-消費者 | 生產者和消費者搶緩衝區 | 訊號量 |
| 哲學家用餐 | 大家都在等叉子 | 奇偶策略 |

## 習題 6：行程與檔案

### 這在做什麼？

學習 Linux/Unix 的**行程 (Process)** 和**檔案操作**系統呼叫。

### 為什麼要學這個？

```
這是 Linux 系統程式的基礎！
═══════════════════════════════
- 所有的程式都是行程
- 所有資料都來自檔案
- 了解這些才能寫系統工具

學會後可以做：
✓ 自己寫 shell (命令列介面)
✓ 行程管理工具
✓ 檔案處理工具
```

### fork() - 建立行程

```python
import os

pid = os.fork()  # 複製一個自己

if pid > 0:
    print("我是父行程")
elif pid == 0:
    print("我是子行程")
```

```
比喻：影印機
═══════════════════════════
fork() 就像按影印機：
- 影印出來的是拷貝
- 拷貝和原本之後可以各自修改
- 變成兩個獨立的程式
```

| fork() 返回值 | 意義 |
|---------------|------|
| > 0 (正數) | 在父行程中，這是子行程的 PID |
| = 0 | 在子行程中 |
| = -1 | 失敗了 |

### execvp() - 執行新程式

```python
os.execvp("ls", ["ls", "-l"])
```

```
比喻：換掉工廠裡的機器
═══════════════════════════
execvp 把現在的程式完全換掉：
- 原本的程式碼被清除
- 放進新的程式
- 但還是同一個 PID (同一間工廠)
```

**fork + execvp = 建立新程式**

```
流程：
1. fork() → 建立子行程 (拷貝)
2. 子行程 execvp() → 換成新程式
3. 父行程 wait() → 等待子行程完成
```

### 檔案操作

| 函數 | 功能 | 比喻 |
|------|------|------|
| `os.open()` | 開啟檔案 | 打開抽屜 |
| `os.close()` | 關閉檔案 | 關上抽屜 |
| `os.read()` | 讀取資料 | 看抽屜裡的東西 |
| `os.write()` | 寫入資料 | 把東西放進抽屜 |

```python
# 寫入檔案
fd = os.open("hello.txt", os.O_WRONLY | os.O_CREAT, 0o644)
os.write(fd, b"Hello!")
os.close(fd)

# 讀取檔案
fd = os.open("hello.txt", os.O_RDONLY)
data = os.read(fd, 100)
os.close(fd)
```

### dup2() - 檔案描述符複製

```python
# 把標準輸出重導向到檔案
fd = os.open("output.txt", os.O_WRONLY)
os.dup2(fd, 1)  # 1 = STDOUT_FILENO
```

```
比喻：複製桌號
═══════════════════
dup2(old, new) = 把 new 號碼改成指向 old 的餐廳

原本：1 號桌 (螢幕)
dup2(fd, 1) 之後：1 號桌指向 fd 的檔案

之後 printf() 會寫到檔案而不是螢幕！
```

### 標準檔案描述符

每個程式一開始就有三個標準的檔案：

| 編號 | 名稱 | 預設對象 | 比喻 |
|------|------|----------|------|
| 0 | stdin | 鍵盤 | 餐廳的點餐口 |
| 1 | stdout | 螢幕 | 餐廳的出菜口 |
| 2 | stderr | 螢幕 | 餐廳的抱怨處理 |

## 期中：記憶體安全

### 這在做什麼？

學習什麼是**記憶體安全**，以及如何避免記憶體相關的錯誤和漏洞。

### 為什麼要學這個？

```
記憶體問題是很多程式崩潰和安全漏洞的原因！
═══════════════════════════════════════════
- 歷史上很多重大的病毒攻擊都是記憶體問題
- 了解這些問題才能寫出安全的程式
- 現在很多新語言 (Rust, Go) 都強調記憶體安全
```

### 記憶體的種類

```
記憶體布局：
═══════════════════════════
高位址
┌─────────────────────┐
│       堆疊          │ ← 函數的局部變數
├─────────────────────┤
│        堆           │ ← 動態分配 (malloc)
├─────────────────────┤
│     .bss            │ ← 未初始化
├─────────────────────┤
│     .data           │ ← 已初始化
├─────────────────────┤
│     .text           │ ← 程式碼
└─────────────────────┘
低位址
```

### 常見的記憶體問題

#### 1. 緩衝區溢位 (Buffer Overflow)

```
比喻：籃子只能放 5 個蘋果，你放了 10 個！
═══════════════════════════════════════

在 C 語言中：
char buffer[5];
strcpy(buffer, "Hello World!");  // 太長了！

結果：
- 溢出的資料會覆蓋其他記憶體
- 可能造成程式崩潰
- 可能被駭客利用執行惡意程式碼
```

#### 2. 使用已釋放的記憶體 (Use-After-Free)

```
比喻：房子賣了還繼續住！
═══════════════════════════════

1. malloc(100) → 拿到一塊記憶體
2. 使用這塊記憶體
3. free() → 歸還給系統
4. 再次使用 → 這塊記憶體可能已經被別人拿走了！

結果：
- 可能讀取到錯誤的資料
- 可能造成程式崩潰
- 可能被駭客利用
```

#### 3. 記憶體泄漏 (Memory Leak)

```
比喻：水龍頭沒關，水一直流出去！
═══════════════════════════════

1. malloc(100) → 拿到記憶體
2. 用完了但忘記 free()
3. 記憶體一直被佔用
4. 長期下來記憶體越來越少
5. 最終耗盡，程式崩潰

Python 會自動清理，但 C/C++ 需要手動處理！
```

### 安全語言 vs 不安全語言

| 語言 | 安全性 | 原因 |
|------|--------|------|
| Python | ✅ 非常安全 | 自動記憶體管理、邊界檢查 |
| Java | ✅ 非常安全 | 垃圾回收、邊界檢查 |
| Rust | ✅ 非常安全 | 編譯時檢查記憶體 |
| C | ❌ 不安全 | 手動管理記憶體 |
| C++ | ❌ 不太安全 | 手動管理記憶體 |

### 如何確保記憶體安全？

```
使用安全語言：
═══════════════════════════════
✅ Python、Java、Rust、Go

安全程式設計原則：
═══════════════════════════════
1. 初始化所有指標
2. 檢查指標是否為空
3. 釋放後設為 NULL
4. 檢查邊界
5. 不要重複釋放

使用工具檢測：
═══════════════════════════════
- Valgrind：檢測記憶體泄漏
- AddressSanitizer：檢測緩衝區溢位
- GDB：調試工具
```
